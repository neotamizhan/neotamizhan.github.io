## Writing a post

Every post is a Markdown file in the `_posts/` folder, named with the pattern
`YYYY-MM-DD-a-short-slug.md`. The block at the very top — between the two lines
of `---` — is the *front matter*. It sets the title, date, and tags. Everything
below it is the article body, written in plain Markdown.

To publish, you commit the file and push. GitHub rebuilds the site and the post
appears on the home page automatically, newest first.

## What Markdown gives you

You get **bold**, *italics*, [links](https://siddhuv.com), and lists:

- Bullet points
- Nested ideas
- Quick structure

> Block quotes read like this — useful for pulling out a line that matters.

Inline `code` and fenced blocks both work, with syntax highlighting:

```python
def horizon_scan(signals: list[Signal]) -> Register:
    """Score, dedupe, and age signals into a persistent register."""
    return Register.from_signals(signals).age().dedupe()
```

Headings, images, and horizontal rules all render in the site's type system —
you never touch the CSS to write a post.
